Software para auditar modelos IFC 🔎

Software programado en Python 🐍 que permite comparar la información establecida en un modelo IFC 📋 con la incluida en una hoja de Excel 💭. Comprueba objeto por objeto que los Property Sets estén bien incluidos, mostrando un informe 💹 final con los Global ID de los objetos que no aprueben la auditoría ⛔.
Open BIM para la verificación de la información

Con un programa muy sencillo hecho en Python 🐍 se puede verificar si un modelo tiene cargado correctamente el set de propiedades 📄 que tengamos como plantilla. Échale un ojo al video 👇 En el vídeo se puede ver como el programa genera una hoja de cálculo que indica los objetos que tienen errores ⛔ en los datos. Además hace un muestreo de los objetos que hay en el modelo, indicando cuantos objetos 🔢 hay por subclase de IfcBuildingElement. Open BIM 📂 nos permite tener un Sistema de Gestión de Calidad muy exhaustivo ¿Qué os parece? (Modelo descargado del repositorio abierto de la Universidad de Auckland 🔓)

¿Cómo funciona?

Librerías Usadas

El primer paso es conocer que librerías se necesitan para el programa. Para abrir y extraer el contenido de modelos IFC 🏙 uso IfcOpenShell. Para trabajar con ficheros Excel uso openpyxl 🧮

Excel de entrada.

Para comprobar que todos los objetos cumple ✅ con los requerimientos de información establecidos, el programa tiene que conocer la información 💽 que debería estar incluido. Para ello, se rellena una hoja excel con este formato 👇:

Ejemplo de uso.

En el caso de ejemplo, busco comprobar que este Set de Propiedades 👇 se encuentren en todos los objetos 🚪 del modelo. Este puede ser el set de propiedades establecido por el EIR de una administración 🌍 o de una empresa 💰. Con este programa y con la metodología BIM, en concreto openBIM 🔓 se pueden simplificar el flujo de trabajo de auditoría verda

El programa lee 📖 esta información del fichero y va objeto por objeto comprobando ❓ que existe este esquema. En el caso de que no se encuentre la información ❌, apunta el GUID apun del objeto para mostrar esta información en el fichero de resultados 📃

Una vez terminado el proceso, genera un archivo Excel 📇 que muestra una lista de GUID's de los objetos que no cumple ❌ con el estándar de datos. Además muestra todos los tipos de clases IFC encontrados 🔍, junto con el numero de objetos 🔢 que hay por clase. Mostrando este dato también en un gráfico 📊
